## Endpoints

### Get Client Details
`GET /api/clients/[id]`

Fetches comprehensive client details including all related data (care requests, shifts, patterns, etc.).

**Logic:** Retrieves client by ID, then fetches all related data including:
- Care requests for the client
- Care patterns (recurring schedules)
- Care exceptions
- Shift assignments with caregiver details
- Preferred caregivers (resolved from UUID array)
- Blacklisted caregivers (resolved from UUID array)
- Calculated statistics

**Expects:** Client ID as URL parameter
